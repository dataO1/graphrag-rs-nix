# claude-code-memory (asset bundle)

Internal asset bundle for Claude Code's long-term memory feature.
Contains:

- **Skills** (`skills/`):
  - `consolidate-memory` — distil findings + catch up missed log
    rows.
  - `recall-and-think` — multi-hop deep recall. User-invoked.
  - `document-decision` — capture a decision with alternatives,
    rationale, rollout/rollback. Auto-invoked on user choices.
  - `log-session-action` — append a row to today's session log
    after a meaningful unit of work. Auto-invoked.
- **CLAUDE.md** — compressed always-on guidance (~15 lines after
  the trigger rules + structural-abstention check).

## How this is loaded

NOT via Claude Code's plugin format. This bundle is consumed by
the `programs.claude-code-memory` home-manager module
(`modules/claude-code.nix` in graphrag-rs-nix), which wires the
individual paths into the upstream `programs.claude-code.*`
options:

- `programs.claude-code.skills.<name>` ← each `skills/<name>` dir
- `programs.claude-code.memory.source` ← `CLAUDE.md`
- `programs.claude-code.mcpServers.memory` ← memory-mcp registration
  (host-baked URL/sessionId)
- `programs.claude-code.settings.hooks.UserPromptSubmit` ← staleness
  check (host-baked, generated by `passthru.mkStalenessHook`)

The user enables `programs.claude-code.enable = true` (upstream
home-manager module) plus `programs.claude-code-memory.enable = true`
(this bundle's wiring module). Claude Code itself is installed and
managed by the upstream module — no wrapping, no `--plugin-dir`,
no `/etc/claude-code/managed-*.json` shenanigans.

Earlier iterations tried Claude Code's plugin format
(`.claude-plugin/plugin.json` + `--plugin-dir <path>`). Anthropic's
plugin loader has no declarative discovery mechanism — `--plugin-dir`
is the only documented path, which forced binary wrapping and
collided with home-manager's standard claude-code install. Wiring
through `programs.claude-code.*` is cleaner and uses upstream-blessed
mechanisms throughout.

## Internal-only

Not for distribution. Bumping graphrag-rs-nix's flake input rolls
the MCP server, hooks, skills, and prompt guidance forward in
lockstep on every `nixos-rebuild switch`.
